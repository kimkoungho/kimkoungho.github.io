## AutoEncoder 와 ML 엔진을 이용한 Anomaly Detection

### AutoEncoder
Encoder 와 Decoder 로 구성
Encoder : 입력데이터를 압축된 형태의 데이터 생성
Decoder : 압축된 데이터를 재구축하여 결과를 추출

score : encoder 의 입력데이터와 decoder 의 결과 데이터 diff

### Anomaly - 비정상 케이스 
데이터가 희소한 경우들 ...

- 레이블이 없음 ! (AutoEncoder 를 선택한 이유 - 비지도학습)


1. User Segmentation 모듈 : 유저의 성향을 정의해서 세그먼트를 나눔
- seg1 : 결제 없는 유저
- seg2 : 최근 2일간 결제 없는 유저
- seg3 : 최근 2일간 만원정도 결제
- seg4 : 최근 2일간 만원이상 결제

2. Traing을 위한 전처리
- 윈도우를 어떻게 동작시키는지에 따라 성능 차이가 남 (슬라이딩 윈도우 방식을 이용했음)

## 딥러닝의 기저, 함수, 근사화 그 활용

### 함수 근사화 능력 (Universal Approximator)
DNN 의 함수 근사화 능력 : 어떤 함수의 내부 내용을 모르더라도 입력과 출력을 이용해서 함수를 근사화할 수 있다!

입력 : 제어 데이터
출력 : 센서 데이터 (예측하려고 하는 값)

Conditional GAN 을 이용해서 과거의 데이터를 이용하여 미래의 데이터를 예측하는 모델을 생성
- AutoEncoder 를 이용하는 것이 일반적인 방법임
